# ACICP Fast-Track Bridge Course

**International Cybersecurity & Digital Forensics Academy (ICDFA)**  
**2-Week Intensive Pre-Programme Module**

---

## Overview

The **ACICP Fast-Track Bridge Course** is a 2-week intensive module designed for applicants who meet most but not all admission requirements. This course ensures that all trainees start the main ACICP programme on a level playing field with strong foundational knowledge.

### Who Should Take the Bridge Course?

The Bridge Course is recommended for applicants who:

- Have cybersecurity knowledge but lack formal certification (CompTIA Security+)
- Have networking experience but need TCP/IP refresher
- Have Linux exposure but lack command-line proficiency
- Have been away from technical work and need to refresh skills
- Are transitioning from non-technical backgrounds into cybersecurity

### Course Duration & Intensity

- **Duration:** 2 weeks (10 business days)
- **Schedule:** 8 hours per day, 5 days per week (40 hours total)
- **Format:** Instructor-led with hands-on labs
- **Prerequisite:** Acceptance into the Bridge Course by ICDFA admissions

---

## Course Objectives

Upon completion of the Bridge Course, trainees will be able to:

1. **Cybersecurity Fundamentals**
   - Explain core security principles (CIA triad, defense in depth, zero trust)
   - Identify common threats and vulnerabilities
   - Understand security frameworks and compliance standards
   - Apply security best practices to real-world scenarios

2. **Networking Essentials**
   - Understand TCP/IP model and protocols
   - Configure and troubleshoot network connectivity
   - Analyze network traffic using packet analysis tools
   - Understand firewalls, routing, and network segmentation

3. **Linux Proficiency**
   - Navigate the Linux filesystem and file permissions
   - Execute essential Linux commands and scripts
   - Manage users, groups, and system services
   - Perform basic system administration tasks

4. **Hands-On Lab Skills**
   - Set up virtual lab environments
   - Execute practical security exercises
   - Document and troubleshoot technical issues
   - Collaborate with peers on technical challenges

## Curriculum Outline

### Week 1: Cybersecurity & Networking Foundations

#### Day 1: Cybersecurity Fundamentals
- **Topics:**
  - History and evolution of cybersecurity
  - CIA Triad (Confidentiality, Integrity, Availability)
  - Defense in Depth and Zero Trust Architecture
  - Common threats and attack vectors
  - Security frameworks (NIST, ISO 27001)

- **Labs:**
  - Lab 01: Identifying security threats in case studies
  - Lab 02: Designing a zero trust network architecture

- **Expected Outcomes:**
  - Understand fundamental security concepts
  - Recognize common vulnerabilities
  - Apply security principles to design problems

#### Day 2: Network Fundamentals
- **Topics:**
  - OSI Model and TCP/IP Model
  - IP addressing and subnetting
  - DNS, DHCP, and network services
  - Routing and switching basics
  - Network security concepts

- **Labs:**
  - Lab 03: Subnetting and IP addressing exercises
  - Lab 04: Configuring DHCP and DNS services
  - Lab 05: Analyzing network topology

- **Expected Outcomes:**
  - Understand network architecture
  - Configure basic network services
  - Analyze network communications

#### Day 3: Packet Analysis & Network Security
- **Topics:**
  - Packet structure and analysis
  - Common protocols (HTTP, HTTPS, SSH, FTP)
  - Network traffic analysis tools (Wireshark, tcpdump)
  - Identifying malicious traffic
  - Firewall rules and access control lists (ACLs)

- **Labs:**
  - Lab 06: Capturing and analyzing packet traces
  - Lab 07: Identifying protocol anomalies
  - Lab 08: Configuring firewall rules

- **Expected Outcomes:**
  - Capture and analyze network traffic
  - Identify suspicious network activity
  - Configure basic firewall security

#### Day 4: Linux Fundamentals Part 1
- **Topics:**
  - Linux filesystem hierarchy
  - File permissions and ownership
  - Basic commands (ls, cd, pwd, mkdir, rm, cp, mv)
  - Text editors (vi, nano)
  - Command-line navigation and shortcuts

- **Labs:**
  - Lab 09: Navigating the Linux filesystem
  - Lab 10: Managing files and directories
  - Lab 11: Understanding file permissions

- **Expected Outcomes:**
  - Navigate Linux filesystem confidently
  - Manage files and permissions
  - Use text editors effectively

#### Day 5: Linux Fundamentals Part 2
- **Topics:**
  - User and group management
  - System services and processes
  - Package management (apt, yum)
  - System logging and monitoring
  - Basic shell scripting

- **Labs:**
  - Lab 12: Creating users and groups
  - Lab 13: Managing system services
  - Lab 14: Writing simple bash scripts

- **Expected Outcomes:**
  - Manage Linux users and permissions
  - Control system services
  - Write basic shell scripts

### Week 2: Hands-On Integration & Capstone

#### Day 6: Integrated Lab Environment Setup
- **Topics:**
  - Setting up virtual machines and containers
  - Configuring lab networks
  - Installing security tools
  - Connecting to lab infrastructure
  - Troubleshooting common issues

- **Labs:**
  - Lab 15: Deploying virtual machines
  - Lab 16: Configuring network connectivity
  - Lab 17: Installing security tools (Nmap, Wireshark, Metasploit)

- **Expected Outcomes:**
  - Set up lab environments independently
  - Configure network connectivity
  - Install and use security tools

#### Day 7: Security Tools & Practical Application
- **Topics:**
  - Network reconnaissance tools (Nmap, Shodan)
  - Vulnerability scanning (OpenVAS, Nessus)
  - System hardening basics
  - Log analysis and monitoring
  - Incident detection

- **Labs:**
  - Lab 18: Network scanning with Nmap
  - Lab 19: Vulnerability scanning
  - Lab 20: Analyzing system logs

- **Expected Outcomes:**
  - Perform network reconnaissance
  - Identify vulnerabilities
  - Analyze security logs

#### Day 8: Offensive & Defensive Basics
- **Topics:**
  - Common attack techniques
  - Exploitation frameworks
  - Defense mechanisms and countermeasures
  - Incident response basics
  - Forensic fundamentals

- **Labs:**
  - Lab 21: Basic exploitation exercises
  - Lab 22: Implementing defensive controls
  - Lab 23: Incident response walkthrough

- **Expected Outcomes:**
  - Understand common attacks
  - Implement basic defenses
  - Respond to security incidents

#### Day 9: Capstone Project Part 1
- **Topics:**
  - Capstone project overview and requirements
  - Research and planning
  - Technical design and architecture
  - Team collaboration

- **Project:**
  - Design a secure network for a fictional organization
  - Identify threats and vulnerabilities
  - Propose security controls and mitigations
  - Document design decisions

- **Expected Outcomes:**
  - Apply learned concepts to real-world scenario
  - Work effectively in teams
  - Document technical designs

#### Day 10: Capstone Project Part 2 & Assessment
- **Topics:**
  - Capstone project completion
  - Presentation and defense
  - Final assessment
  - Transition to main ACICP programme

- **Project:**
  - Complete capstone project
  - Present findings to instructors
  - Defend design decisions
  - Receive feedback

- **Assessment:**
  - Practical lab exercises (50%)
  - Capstone project (30%)
  - Participation and engagement (20%)
  - **Passing score: ≥70%**

- **Expected Outcomes:**
  - Complete capstone project successfully
  - Demonstrate mastery of bridge course material
  - Prepare for main ACICP programme

## Transition to Main ACICP Programme

### Upon Successful Completion

Trainees who successfully complete the Bridge Course with a score of ≥70% will:

- Receive a **Bridge Course Completion Certificate**
- Be formally admitted to the main ACICP programme
- Begin the ACICP on the following Monday
- Have access to all ACICP resources and support services

### If You Do Not Pass

Trainees who score below 70% on the Bridge Course:

- Will not be admitted to the main ACICP programme
- May reapply for the next Bridge Course cohort (3 months later)
- Will receive feedback on areas for improvement
- May receive recommendations for additional study resources

## FAQ

**Q: Can I skip the Bridge Course and go directly to the main ACICP?**  
A: Only if you meet all admission requirements. The Bridge Course is optional for those who need additional preparation.

**Q: What if I fail the Bridge Course?**  
A: You will not be admitted to the main ACICP. You may reapply for the next Bridge Course cohort.

**Q: Is there additional cost for the Bridge Course?**  
A: The Bridge Course is included in your ACICP tuition. No additional fees apply.

**Q: Can I work while taking the Bridge Course?**  
A: The Bridge Course is full-time (40 hours per week). Part-time employment is not recommended.

**Q: What if I have prior experience in some topics?**  
A: The Bridge Course is designed to ensure everyone has the same foundational knowledge. All trainees must complete all modules.
